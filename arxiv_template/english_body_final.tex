\begin{abstract}
Automatic recognition of circuit schematic diagrams is a core challenge in Electronic Design Automation (EDA), involving visual text extraction and topological graph tracing. This paper presents CircuitOCR, a circuit schematic OCR system based on PaddleOCR-VL-0.9B. We construct the V5 Golden multi-source balanced dataset, comprising 1,857 KiCad schematics (500 synthetic + 1,357 real-world open-source) and 698 Masala-CHAI textbook circuits, totaling 2,555 samples after SPICE artifact cleaning and stratified sampling. Through six controlled experiments (E1--E6), we identify the \textbf{root cause of modality collapse}: LoRA fine-tuning of the vision-language projector destroys pre-trained alignment weights, causing the LLM to degenerate into repetitive output. The proposed LLM-Only LoRA architecture completely resolves collapse (diversity 4\% $\to$ 90\%). The final V8-Fixed model, with wide LoRA ($r=16$, 5.7M params) plus causal token shift and BPE boundary fixes, achieves Avg.\ NED = 0.7760 on easy100 (baseline 0.9372, 17.2\% error reduction) and 0.8257 on easy50 validation (baseline 0.9634, 14.3\% error reduction). Training completes in $\sim$2 hours on a single RTX 4060 8\,GB. Model weights, datasets, and online demo are fully open-sourced.
\end{abstract}

\section{Introduction}

\subsection{Background}

Vision-Language Models (VLMs) have demonstrated significant advantages in document parsing, directly outputting structured text from images end-to-end. However, automatic circuit schematic recognition remains research-scarce---no dedicated public benchmark exists. Existing datasets like Open Schematics~\citep{openschematics2025} and Masala-CHAI~\citep{bhandari2025masalachai} provide circuits and netlists but lack standardized evaluation and suffer from severe annotation noise.

The global semiconductor market exceeded \$600B in 2025 (SEMI), with PCB design tools at \~\$4B. Reverse engineering, legacy digitization, and cross-tool migration drive demand; the PCB reverse engineering outsourcing market alone exceeds \$500M annually. CircuitOCR fills this gap.

\subsection{Core Contributions}

\begin{enumerate}[nosep]
    \item \textbf{V5 Golden Dataset}: First format-aligned, zero-AI-label, multi-source balanced circuit OCR benchmark (2,555 samples);
    \item \textbf{Root cause of modality collapse}: E1--E6 experiments identify Projector fine-tuning as the mechanism; LLM-Only LoRA resolves it (diversity 4\% $\to$ 90\%);
    \item \textbf{V8-Fixed model}: easy100 NED 0.7760 (baseline 0.9372, 17.2\% error reduction), easy50 NED 0.8257 (baseline 0.9634, 14.3\% error reduction);
    \item \textbf{Complete open-source ecosystem}: Dataset, weights, and demo all publicly released.
\end{enumerate}


\section{Background and System Setup}

\subsection{Base Model and Task Definition}

We use PaddleOCR-VL-0.9B~\citep{cui2025paddleocrvl} (908M params), integrating a NaViT dynamic-resolution vision encoder with ERNIE-4.5-0.3B LLM. Circuit schematic understanding comprises two sub-tasks: (1) \textbf{Visual text recognition}---extracting component designators (R1, C1), values (10k, 100nF), net labels (VCC/GND); (2) \textbf{Topology graph tracing}---inferring electrical connectivity from images. We use a dual-layer evaluation protocol: text NED and topology F1.

\subsection{Base Model Failure Modes}

The unfine-tuned base model exhibits three failure modes (Fig.~\ref{fig:base_failures}): (1) repetitive token degeneration; (2) memorized hallucination (generic pre-training text); (3) partial recognition with omission. On easy100, base Avg.\ NED = 0.9372; on easy50 validation subset (10 samples), base Avg.\ NED = 0.9634.

\section{V5 Golden Dataset}

\subsection{Three Fatal Flaws in Legacy Datasets}

V1--V4 datasets had severe quality issues:

\begin{enumerate}[nosep]
    \item \textbf{GT-image misalignment}: GT from KiCad S-expression parsing included invisible text (pin numbers, hidden net names);
    \item \textbf{Format mismatch}: Open Schematics/Rescraped used horizontal space-concatenated format vs.\ test set's one-word-per-line vertical---V4 had only 62.1\% single-word lines vs.\ test set's 98.6\%;
    \item \textbf{Duplication noise}: 22,340 intra-line adjacent duplicate pairs, contaminating 24.3\% of samples.
\end{enumerate}

\subsection{Three Design Principles}

\textbf{Principle 1: 100\% GT-image alignment.} Synthetic from \texttt{draw.text()} recording; real KiCad from SVG stroked-text extraction; Masala from SPICE netlists. All deterministic, zero AI labeling.

\textbf{Principle 2: Multi-source balance.} Masala-CHAI downsampled from 4,858 to 698, 0.38:1 ratio vs.\ 1,857 KiCad, preventing SPICE format dominance.

\textbf{Principle 3: Test set format alignment.} Removed Open Schematics (226) and Rescraped (102); replaced with train-real (1,357). Format consistency improved from 62.1\% to 97.9\%; duplication noise reduced 99.5\%.

\subsection{Three Data Sources}

\textbf{Synthetic V3 (500).} Random circuit topologies (5--30 components, 10 types), PNG rendering, GT from \texttt{draw.text()} logs~\citep{zhang2026circuitocr}.

\textbf{train-real (1,357).} GitHub/KiCad open-source community, kicad-cli SVG $\to$ cairosvg PNG, GT from SVG stroked-text visible elements, one-word-per-line vertical format.

\textbf{Masala-CHAI (698).} $\sim$7,500 textbook circuits with SPICE netlists~\citep{bhandari2025masalachai}. Four-category artifact cleaning; 4,858 $\to$ 4,421 clean; quality-scored stratified sampling to 698.

\section{Model Fine-Tuning and Performance}

\subsection{Discovery of Modality Collapse (E1--E6)}

Initial LoRA experiments ($r=8$, q/k/v/o self-attention) on legacy datasets observed extremely low output diversity---nearly all test samples produced identical ``12\textbackslash n100\textbackslash n100...'' sequences (Fig.~\ref{fig:old_collapse}). We define this as \textbf{Modality Collapse}.

\begin{figure}[h]
\centering
\includegraphics[width=0.7\textwidth]{../circuit-ocr-dataset/figures/v5_NMOS_Circuit_2_old.png}
\caption{Old model (V4, +Projector LoRA $r=16$) output: all 50 test samples produce identical ``12 100 100...'', 4\% diversity---classic modality collapse.}
\label{fig:old_collapse}
\end{figure}

Six controlled experiments systematically identified the root cause (Table~\ref{tab:ablation}):

\begin{table}[h]
\centering
\caption{Modality Collapse Root Cause Diagnosis (E1--E6)}
\label{tab:ablation}
\begin{tabular}{lcp{2.5cm}p{3.5cm}}
\toprule
\textbf{Experiment} & \textbf{Variable} & \textbf{Result} & \textbf{Conclusion} \\
\midrule
E1: Blank image  & Real vs.\ gray    & Outputs differ & Vision encoder works \\
E2: Resolution   & 168px $\to$ 336px & Still collapses & Resolution not root cause \\
E3: Epoch count  & 1 vs.\ 3 epochs   & NED improves   & Positive but insufficient \\
E4: LoRA layers  & Add Projector     & NED improves   & \textbf{Projector is key bottleneck} \\
E5: LoRA rank    & $r=8 \to r=16$   & NED improves   & Capacity helps, diversity still 4\% \\
E6: LLM-Only     & Freeze Projector  & Diversity 90\% & \textbf{Collapse fully resolved} \\
\bottomrule
\end{tabular}
\end{table}

\textbf{Core finding: Projector fine-tuning is the collapse root cause.} The Projector (\texttt{mlp\_AR}, 25.9M params), as the sole vision$\to$language bridge, when perturbed by LoRA, outputs out-of-distribution vectors. The LLM degenerates into mechanically repeating the highest-frequency token sequence. Freezing the Projector to protect pre-trained alignment while teaching only the LLM to reinterpret existing visual features is the safest strategy.

\subsection{V5: LLM-Only LoRA Validation}

V5 freezes Projector, fine-tunes only LLM self-attention ($r=8$, 1.25M params), 384px resolution, $2\times10^{-5}$ LR. After 1 epoch on 1,857 samples (928 steps, 22 min): 90\% diversity---the model for the first time produces different outputs for different images (Fig.~\ref{fig:v5_output}). But $r=8$ capacity insufficient; premature convergence at $\sim$200 steps, NED below baseline---pointing the way for V8.

\subsection{V8-Fixed: Three Key Breakthroughs}

V8-Fixed inherits LLM-Only architecture, training on V5 Golden (2,299 samples) for 3 epochs (1,800 steps).

\textbf{Breakthrough 1: Wide LoRA capacity ($r=16$, 5.7M).} Rank doubled, target modules expanded from 153 to 310 projection matrices. Loss converged from 2.71 to 0.30 with no collapse.

\textbf{Breakthrough 2: Causal token shift fix.} Earlier scripts double-shifted PaddleOCR-VL's already-shifted logits, corrupting gradient signals. V8-Fixed implements correct single-shift.

\textbf{Breakthrough 3: BPE boundary merging fix.} Prompt and label tokenized independently then concatenated in token-ID space, eliminating BPE cross-boundary merging (e.g., ``\textbackslash nR'' $\to$ single token) that swallowed label starting characters.

\section{Experimental Setup}

\subsection{Training Hyperparameters}

All experiments on single RTX 4060 8\,GB. V8-Fixed final configuration:

\begin{itemize}[nosep]
    \item LoRA: $r=16$, $\alpha=32$, 310 matrices (LLM + vision encoder + projection), 5.7M params (0.63\%)
    \item Optimizer: AdamW ($\beta_1=0.9$, $\beta_2=0.95$, decay=0.1)
    \item LR: $2\times10^{-5}$, cosine annealing, 3 epochs (1,800 steps), batch\_size=1, grad\_accum=4
    \item max\_dim: 384px, label truncation: 200 chars, training $\sim$2 hours
\end{itemize}

\subsection{Evaluation Metrics}

Avg.\ NED: $\text{NED}(p, g) = \text{Levenshtein}(p, g) / \max(|p|, |g|)$. \texttt{--unordered} mode sorts lines before comparison. Easy50 uses 10-sample validation subset; easy100 uses 100-sample full test set. Diversity = unique outputs / total samples.


\section{Discussion}

\subsection{Modality Collapse Mechanism}

Projector perturbed by LoRA $\to$ visual features distorted into OOD vectors $\to$ LLM attention degenerates $\to$ autoregression collapses to highest-frequency token repetition. LLM-Only LoRA completely avoids this by freezing the Projector.

\subsection{Data Quality $>$ Data Quantity}

V4's 5,686 samples with format mismatch (62.1\% vs.\ 98.6\%) and 22,340 duplication pairs severely corrupted training. V5 Golden's 2,555 cleaner samples produced V8-Fixed's 0.7760 NED---clean small datasets outperform noisy large datasets.

\subsection{The NED--Diversity Trade-off}

% diversity---genuinely reading images. Structured-output tasks require dual NED + diversity evaluation.


\section{Conclusion}

\subsection{Key Contributions}

\begin{enumerate}
    \item \textbf{V5 Golden Dataset}: 2,555 samples, 100\% deterministic GT, 98.6\% format-aligned with test set, 99.5\% noise reduction;
    \item \textbf{Modality collapse root cause}: E1--E6 experiments identify Projector fine-tuning as the mechanism; LLM-Only LoRA restores diversity from 4\% to 90\%;
    \item \textbf{V8-Fixed model}: easy100 NED 0.7760 (baseline 0.9372, 17.2\% error reduction), easy50 NED 0.8257 (baseline 0.9634, 14.3\% error reduction)---current state-of-the-art;
    \item \textbf{Complete open-source release}: Dataset, weights, and demo all publicly available.
\end{enumerate}

\subsection{Limitations and Outlook}

Current limitations: RTX 4060 8\,GB (no Flash Attention), topology evaluation protocol designed but unimplemented. Future work: (1) implement topology evaluation (component F1, pin accuracy); (2) scale to larger GPUs with Flash Attention; (3) integrate SPICE simulation verification as reward signal; (4) migrate LLM-Only LoRA to larger VLMs.
