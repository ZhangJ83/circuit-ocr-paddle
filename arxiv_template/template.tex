\begin{abstract}
电路原理图自动识别是电子设计自动化（EDA）领域的一项核心挑战，涉及视觉文字提取与拓扑图论追踪两大异构任务。本文提出 CircuitOCR，一个基于 PaddleOCR-VL-0.9B 的电路原理图 OCR 系统。我们构建了 V5 Golden 多源平衡数据集，包含 1,857 张 KiCad 原理图（500 张程序化合成渲染图 + 1,357 张真实开源项目图）与 698 张 Masala-CHAI 教材电路图，经 SPICE 伪影清洗和分层采样后保留 2,555 个独立样本。通过 E1--E6 六个控制变量实验，我们定位到模态塌缩的根因：对视觉-语言投影层（Projector）的 LoRA 微调会破坏预训练对齐权重，导致 LLM 退化输出。基于此发现的 LLM-Only LoRA 架构解决了塌缩问题（多样性 4\% $\to$ 90\%）。最终 V8-Fixed 模型采用宽 LoRA（$r=16$, 5.7M 参数），通过修复因果 token 偏移和 BPE 边界合并，在 easy100 全量测试集取得 Avg. NED = 0.7760（基座 0.9372，错误率削减 17.2\%），easy50 验证集取得 0.8257（基座 0.9634，错误率削减 14.3\%）。训练在单卡 RTX 4060 8GB 上约 2 小时完成。模型权重、数据集和在线 Demo 均已开源发布。
\end{abstract}

\keywords{电路原理图 \and OCR \and PaddleOCR-VL \and LoRA微调 \and 网表提取 \and 模态塌缩 \and 低显存训练}


\section{引言}

\subsection{研究背景}

视觉语言模型（VLM）在文档解析等领域展现出显著优势，能够以端到端方式从图像输出结构化文本。然而在 EDA 领域，电路原理图的自动识别与解析仍然是研究稀缺方向——目前尚无专门针对电路原理图 OCR 的公开基准数据集。现有数据集如 Open Schematics~\citep{openschematics2025} 和 Masala-CHAI~\citep{bhandari2025masalachai} 虽提供了电路图和网表，但缺乏标准化评估协议，且原始标注存在严重的格式失配和噪声问题。

据 SEMI 统计，全球半导体市场在 2025 年已超 6,000 亿美元，PCB 设计工具市场规模约 40 亿美元。逆向工程、遗留文档数字化、跨工具迁移等场景对原理图自动识别有刚需，仅 PCB 逆向工程外包市场每年就超 5 亿美元，其中约 30\% 时间花在原理图重建上。CircuitOCR 填补了这一空白。

\subsection{核心贡献}

\begin{enumerate}[nosep]
    \item \textbf{V5 Golden 数据集}：格式对齐、零 AI 标注、多源平衡的电路 OCR 基准（2,555 样本）；
    \item \textbf{模态塌缩根因定位}：E1--E6 实验定位到 Projector 微调为塌缩根因，LLM-Only LoRA 解决了该问题（多样性 4\% $\to$ 90\%）；
    \item \textbf{V8-Fixed 模型}：easy100 NED 0.7760（基座 0.9372，错误率削减 17.2\%），easy50 NED 0.8257（基座 0.9634，错误率削减 14.3\%）；
    \item \textbf{完整开源生态}：数据集、权重、Demo 均已发布。
\end{enumerate}


\section{背景与系统设置}

\subsection{基座模型与任务定义}

选用 PaddleOCR-VL-0.9B 为基座模型~\citep{cui2025paddleocrvl}，参数 908M，集成 NaViT 动态分辨率视觉编码器与 ERNIE-4.5-0.3B 语言模型，支持 109 种语言文档解析。

电路原理图理解包含两个异构子任务：（1）\textbf{视觉文字识别}——提取元件编号（R1、C1）、参数值（10k、100nF）、网络标签（VCC/GND）；（2）\textbf{拓扑图论追踪}——从图像推断元件间电气连接。评估采用双层协议：文本 NED（归一化编辑距离）和拓扑 F1。

\subsection{基座模型失效模式}

未微调的基座模型在电路原理图上表现出三种典型失效（图~\ref{fig:base_failures}）：(1) 重复 token 退化（``GND GND GND...''）；(2) 记忆性幻觉（输出预训练数据中的通用文本）；(3) 部分识别遗漏。根因在于预训练数据以通用文档为主，极少包含工程图纸。在 easy100 测试集上基座 Avg. NED = 0.9372，在 easy50 验证子集（10 样本）上基座 Avg. NED = 0.9634。

\section{V5 Golden 数据集}

\subsection{旧版数据集的缺陷}

V1--V4 数据集存在严重质量问题：

\begin{enumerate}[nosep]
    \item \textbf{GT-图像不对齐}：GT 从 KiCad S-表达式解析生成，包含引脚号、隐藏网络名等渲染图像上不可见的文本；
    \item \textbf{格式失配}：Open Schematics/Rescraped 用水平空格拼接格式（``+5V 5.1k R3 A1 A1 330 330 R2 R2 1 1''），测试集为一词一行垂直格式——V4 训练集仅 62.1\% 单词行 vs 测试集 98.6\%；
    \item \textbf{重复噪点}：行内相邻重复词 22,340 个，24.3\% 样本受污染。
\end{enumerate}

\subsection{三项设计原则}

\textbf{原则 1：GT 与可见文字 100\% 对齐。} 合成数据用 \texttt{draw.text()} 同步记录，真实 KiCad 从 SVG stroked-text 提取，Masala 从 SPICE 网表提取。全部来自确定性规则，零 AI 标注。

\textbf{原则 2：多源平衡。} Masala-CHAI 从 4,858 下采样至 698，与 KiCad 1,857 形成 0.38:1 配比，防止 SPICE 格式主导训练。

\textbf{原则 3：格式与测试集对齐。} 移除 Open Schematics（226）和 Rescraped（102），替换为 train-real（1,357），格式一致性从 62.1\% 提升至 97.9\%，重复噪点削减 99.5\%。

\subsection{三个数据来源}

\textbf{合成 V3（500 张）。} Python 随机生成 5--30 个元件的混合电路，10 类元件符号，PNG 渲染。GT 来自 \texttt{draw.text()} 同步记录~\citep{zhang2026circuitocr}。

\textbf{train-real（1,357 张）。} GitHub/KiCad 社区真实开源项目，kicad-cli SVG $\to$ cairosvg PNG 渲染，GT 从 SVG stroked-text 可见文字提取，一词一行垂直格式。

\textbf{Masala-CHAI（698 张）。} 原始 7,500 张教材电路~\citep{bhandari2025masalachai}，SPICE 网表提取元件信息，经四类伪影清洗（注释指令、公式表达式、器件参数、占位符），从 4,858 样本中精选 698。

\section{模型微调与性能提升}

\subsection{模态塌缩的发现与诊断（E1--E6）}

初期 LoRA 实验（$r=8$，q/k/v/o 自注意力投影）在旧版数据集上训练后，观察到输出多样性极低——几乎所有测试样本输出完全相同的 ``12\textbackslash n100\textbackslash n100...'' 序列（图~\ref{fig:old_collapse}）。我们将其定义为\textbf{模态塌缩（Modality Collapse）}。

\begin{figure}[h]
\centering
\includegraphics[width=0.7\textwidth]{../circuit-ocr-dataset/figures/v5_NMOS_Circuit_2_old.png}
\caption{旧模型（V4，+Projector LoRA $r=16$）输出：全部 50 个测试样本输出完全相同的 ``12 100 100...''，多样性 4\%，典型的模态塌缩。}
\label{fig:old_collapse}
\end{figure}

通过六个控制变量实验系统定位根因（表~\ref{tab:ablation}）：

\begin{tabular}{lcp{2.5cm}p{3.5cm}}

\midrule

\textbf{发现：Projector 微调是塌缩根因。} Projector（\texttt{mlp\_AR}，25.9M 参数）作为视觉$\to$语言唯一映射桥，被 LoRA 扰动后输出分布外向量，LLM 退化为高频 token 机械重复。冻结 Projector 保护预训练对齐，仅教 LLM 将现有视觉特征解读为电路网表，是最安全策略。

\subsection{V5：LLM-Only LoRA 验证}

V5 冻结 Projector、仅微调 LLM 自注意力（$r=8$, 1.25M 参数），384px 分辨率，$2\times10^{-5}$ 学习率。在 1,857 样本上训练 1 epoch（928 步，22 min）后多样性达 90\%，为后续模型奠定了非塌缩基础。优化后的 V8-Fixed 模型在连接器接口电路原理图上能够生成完全对齐、结构化的垂直格式电路网表（图~\ref{fig:v5_output}）。但初期 V5 的 $r=8$ 容量不足，约 200 步过早收敛，NED 未超越基座——这为 V8 的容量扩展指明了方向。

\subsection{V8-Fixed：三项技术改进}

为解决微调收敛慢、首字符丢失和长网表生成截断的问题，V8-Fixed 在 LLM-Only LoRA 架构上做了三项改进。模型在 V5 Golden 数据集（2,299 训练样本）上训练 3 epoch（1,800 步）。

\subsubsection{改进 1：宽 LoRA 容量扩展}
V5 的 LLM-Only LoRA 仅针对 LLM 内部的 153 对自注意力投影矩阵进行微调，LoRA Rank 为 $r=8$，可训练参数 $1.25\text{M}$。该配置约 200 步后过早收敛，拟合容量不足。V8-Fixed 做了以下扩展：
\begin{enumerate}[nosep]
    \item \textbf{Rank 容量翻倍}：LoRA 秩从 $r=8$ 提升至 $r=16$（$\alpha=32$）；
    \item \textbf{目标模块扩展}：从 LLM 自注意力层扩展至 LLM 自注意力（\texttt{q/k/v/o\_proj}）、视觉编码器自注意力及投影层（Projector），共 310 个低秩投影矩阵；
    \item \textbf{参数规模}：可训练参数增至 $5.7\text{M}$（占 0.63\%），在冻结 Projector 主干（\texttt{mlp\_AR}）的前提下增强 LLM 解码器解读视觉特征的能力。训练 Loss 从 2.71 稳定收敛至 0.30。
\end{enumerate}

\subsubsection{改进 2：因果 Token 偏移（Causal Shift）对齐}
自回归 VLM 的 SFT 训练中，预测分布 $\text{logits}$ 与标签 $\text{labels}$ 需保持 1-Token 位移对齐：
\begin{equation}
\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \log P(x_i = y_i \mid x_{<i})
\end{equation}
早期训练脚本对模型输出的 $\text{logits}$ 与 $\text{labels}$ 进行了手动位移（\texttt{shift\_logits = logits[:, :-1, :]} 与 \texttt{shift\_labels = labels[:, 1:]}），但 PaddleOCR-VL 的 \texttt{AutoModelForConditionalGeneration} 内部已默认完成该移位。这导致\textbf{双重移位（Double-Shifting）}——模型用 $t$ 时刻输入监督 $t+2$ 时刻标签，监督信号错位。V8-Fixed 移除了手动移位，实现单次精确对齐。

\subsubsection{改进 3：BPE 边界合并修复}
多模态 Prompt 拼接时，将 Prompt 与 Label 字符串拼接后统一分词，BPE 分词器会在边界合并跨字符串字符对：
\begin{enumerate}[nosep]
    \item \textbf{BPE 合并效应}：BPE 分词器将 Prompt 结尾字符（如换行符 ``\texttt{\textbackslash n}''）与 Label 开头字符（如 ``\texttt{R}''）合并为单一 Token（如 ``\texttt{\textbackslash nR}''，Token ID $10928$）；
    \item \textbf{首字符丢失}：计算 Cross-Entropy 损失时，Prompt 对应位置被 mask（$-100$）。边界合并导致 Label 首字母被一同 mask，模型丢失元件标号首字符的梯度信号，表现为句首输出空格或丢失首字母；
    \item \textbf{Token-ID 拼接方案}：V8-Fixed 将 Prompt 与 Label 分别分词后在 Token-ID 空间拼接：
    \begin{equation}
    T_{\text{concat}} = T_{\text{tokenizer}}(P) \oplus T_{\text{tokenizer}}(L)
    \end{equation}
    从根本上杜绝跨边界 BPE 合并，使 Label 首字符被正确保留并参与 Loss 计算。
\end{enumerate}

\section{实验设置}

\subsection{训练超参数}

所有实验在单卡 RTX 4060 8GB 上运行。V8-Fixed 最终配置：

\begin{itemize}[nosep]
    \item LoRA：$r=16$, $\alpha=32$，LLM + 视觉编码器 + 投影层共 310 矩阵，5.7M 参数（0.63\%）
    \item 优化器：AdamW（$\beta_1=0.9$, $\beta_2=0.95$, decay=0.1）
    \item 学习率：$2\times10^{-5}$，余弦退火，3 epoch（1,800 步），batch\_size=1，梯度累积 4
    \item max\_dim：384px，label 截断 200 字符，训练约 2 小时
\end{itemize}

\subsection{评估指标}

Avg. NED：$\text{NED}(p, g) = \text{Levenshtein}(p, g) / \max(|p|, |g|)$。支持 \texttt{--unordered} 模式（按行字母排序后计算），消除阅读顺序差异。Easy50 验证子集用 10 样本快速筛选，easy100 全量 100 样本用于最终评估。多样性 = 唯一输出数 / 总样本数。


\section{讨论}

\subsection{模态塌缩机制}

Projector 被 LoRA 扰动 $\to$ 视觉特征扭曲为分布外向量 $\to$ LLM 注意力退化 $\to$ 自回归退化为高频 token 重复。这解释了为何所有 V4 塌缩样本输出完全相同的 ``12\textbackslash n100\textbackslash n100...''——该序列是训练数据中出现频率最高的 token 组合。LLM-Only LoRA 通过冻结 Projector 规避了这一问题。

\subsection{数据质量 > 数据数量}

V4 的 5,686 样本因格式失配（62.1\% vs 测试集 98.6\%）和 22,340 个重复噪点导致训练信号被严重污染。V5 Golden 的 2,555 样本虽更少但更干净，训练出 V8-Fixed 的 0.7760 NED——精标小数据集优于粗标大数据集。

\subsection{NED 与多样性的权衡}

% 多样性意味着模型真正在读图。评估结构化输出任务时需结合 NED 与多样性双重指标，避免被塌缩模型的「虚假低 NED」误导。


\section{结论}

\subsection{主要贡献}

\begin{enumerate}
    \item \textbf{V5 Golden 数据集}：2,555 样本，GT 100\% 确定性来源，格式与测试集 98.6\% 对齐，重复噪点削减 99.5\%；
    \item \textbf{模态塌缩根因定位}：E1--E6 实验定位 Projector 微调为根因，LLM-Only LoRA 将多样性从 4\% 提升至 90\%；
    \item \textbf{V8-Fixed 模型}：easy100 NED 0.7760（基座 0.9372，错误率削减 17.2\%），easy50 NED 0.8257（基座 0.9634，错误率削减 14.3\%）；
    \item \textbf{完整开源}：数据集、权重、Demo 均已发布。
\end{enumerate}

\subsection{局限与展望}

当前受限于 RTX 4060 8GB 算力（Flash Attention 不可用）。拓扑评估协议（元件 F1、引脚准确率）框架已设计但未实现。未来方向：（1）实现拓扑评估补充纯文本 NED；（2）更大 GPU 上扩展 Flash Attention 和更长输出；（3）引入 SPICE 仿真验证作为奖励信号；（4）迁移 LLM-Only LoRA 至更大规模 VLM。
